4.4 Declarations

The data files contain declarations, that are used to structure the content of the file. These declarations are the header, the IfcProjectLibrary and the IfcProject.

If the file should transport product data, the file shall contain all three parts: header, IfcProjectLibrary and IfcProject. This ensures that the product data stays interpretable over a long time, even if the content of the product data templates have changed during this time (catalogue/template embedding).

The following sections define the details of the three parts of the file.

The data structures contain structures, that are used to declare the content. These declarations are the IfcProjectLibrary and the IfcProject.

For the declaration of templates and type objects the IfcProjectLibrary shall be used. For the declaration of sheets, the IfcProject shall be used.

The cardinality of the classes shall be as following:

Type of exchange IfcProjectLibrary IfcProject
Data templates [1:?] -
Data sheets [1:?] [1:1]
